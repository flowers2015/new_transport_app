# راهنمای رفع مشکل فایل‌های .env

## مشکل
اگر ارتباط با دیتابیس برقرار نمی‌شود، احتمالاً فایل‌های `.env` به درستی ایجاد نشده‌اند.

## راه حل دستی

### 1. ایجاد فایل `.env.development`
در پوشه `frontend` یک فایل با نام `.env.development` ایجاد کنید و این محتوا را در آن قرار دهید:

```
VITE_API_BASE_URL=http://localhost:3000/api/v1
```

### 2. ایجاد فایل `.env.production`
در پوشه `frontend` یک فایل با نام `.env.production` ایجاد کنید و این محتوا را در آن قرار دهید:

```
VITE_API_BASE_URL=/api/v1
```

## نکات مهم

1. **نام فایل باید دقیقاً `.env.development` و `.env.production` باشد** (با نقطه در ابتدا)
2. **فایل‌ها باید در پوشه `frontend` باشند** (هم سطح با `package.json`)
3. **بعد از ایجاد فایل‌ها، سرور Vite را restart کنید:**
   - اگر در حال اجرا است، آن را متوقف کنید (Ctrl+C)
   - سپس دوباره `npm run dev` را اجرا کنید

## بررسی

برای بررسی اینکه فایل‌ها درست ایجاد شده‌اند:

1. در کنسول مرورگر (F12) به تب Console بروید
2. باید پیام‌های دیباگ را ببینید:
   ```
   🔧 [API Config] API_BASE_URL: http://localhost:3000/api/v1
   ```

## اگر هنوز کار نمی‌کند

کد به گونه‌ای نوشته شده که حتی اگر فایل‌های `.env` وجود نداشته باشند، از مقدار پیش‌فرض `http://localhost:3000/api/v1` استفاده می‌کند.

اگر هنوز مشکل دارید:
1. مطمئن شوید که بک‌اند روی پورت 3000 در حال اجرا است
2. در کنسول مرورگر (F12 > Network) بررسی کنید که درخواست‌ها به کدام آدرس ارسال می‌شوند
3. اگر خطای CORS می‌بینید، مطمئن شوید که بک‌اند CORS را برای `http://localhost:5173` فعال کرده است
