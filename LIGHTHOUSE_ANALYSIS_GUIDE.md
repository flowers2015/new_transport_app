# راهنمای تحلیل گزارش‌های Lighthouse - Before/After

## 📊 نحوه تحلیل نتایج Lighthouse

### 1. جمع‌آوری داده‌های Before/After

#### قبل از بهینه‌سازی:
1. Chrome DevTools را باز کنید (F12)
2. به تب **Lighthouse** بروید
3. تنظیمات:
   - **Category**: Performance را انتخاب کنید
   - **Device**: Desktop یا Mobile
   - **Mode**: Navigation
4. روی **Analyze page load** کلیک کنید
5. نتایج را ذخیره کنید یا Screenshot بگیرید

#### بعد از بهینه‌سازی:
- همان مراحل را تکرار کنید
- نتایج را با Before مقایسه کنید

## 🔧 نحوه استفاده از این راهنما

1. **قبل از بهینه‌سازی:**
   - Lighthouse Report بگیرید
   - داده‌های بالا را یادداشت کنید
   - Screenshot از نتایج بگیرید

2. **بعد از بهینه‌سازی:**
   - دوباره Lighthouse Report بگیرید
   - داده‌ها را با Before مقایسه کنید
   - درصد بهبود را محاسبه کنید

3. **تحلیل:**
   - از قالب بالا استفاده کنید
   - بهبودها را مستند کنید
   - نقاط ضعف باقی‌مانده را شناسایی کنید

---

## 📸 نحوه Screenshot گرفتن از Lighthouse

1. Chrome DevTools → Lighthouse
2. بعد از تحلیل، روی **Export Report** کلیک کنید
3. یا از **Screenshot** استفاده کنید
4. فایل JSON را ذخیره کنید برای تحلیل دقیق‌تر

---

## 💡 نکات مهم

1. **شرایط یکسان:**
   - همیشه در همان Device تست کنید
   - Network Throttling را یکسان نگه دارید
   - Cache را Clear کنید قبل از هر تست

2. **چند بار تست:**
   - حداقل 3 بار تست کنید
   - میانگین بگیرید
   - نتایج را مقایسه کنید

3. **تحلیل دقیق:**
   - فقط به Score نگاه نکنید
   - هر متریک را جداگانه تحلیل کنید
   - Bundle Size و Network را هم بررسی کنید

## 🚀 بهبودهای مورد انتظار از بهینه‌سازی‌های انجام شده

### Lazy Loading:
- **FCP**: بهبود 20-30%
- **LCP**: بهبود 15-25%
- **Bundle Size**: کاهش 40-60%

### Code Splitting:
- **TTI**: بهبود 25-35%
- **TBT**: بهبود 30-40%
- **Bundle Size**: کاهش 30-50%

### Memoization:
- **TBT**: بهبود 20-30%
- **Re-render Count**: کاهش 30-50%

### API Caching:
- **Network Requests**: کاهش 50-70%
- **TTI**: بهبود 10-20%
