# 🚀 راهنمای ساده تست سرعت

## مرحله 1: تست با Chrome DevTools (ساده‌ترین روش)

### گام 1: باز کردن اپلیکیشن
1. اپلیکیشن را در Chrome باز کنید
2. Login کنید

### گام 2: باز کردن DevTools
1. کلید **F12** را بزنید (یا راست کلیک → Inspect)
2. در پایین صفحه DevTools باز می‌شود

### گام 3: تست Network (API Calls)
1. روی تب **Network** کلیک کنید
2. مطمئن شوید که **Disable cache** تیک خورده است (برای تست دقیق)
3. صفحه را **Refresh** کنید (F5)
4. منتظر بمانید تا صفحه کامل لود شود
5. در بالای Network Tab این اطلاعات را ببینید:
   - **Requests**: تعداد کل درخواست‌ها (مثلاً 25)
   - **Transferred**: حجم کل دانلود (مثلاً 2.5 MB)
   - **Finish**: زمان کل لود (مثلاً 3.2 s)
6. این اعداد را یادداشت کنید 📝

### گام 4: تست Performance (سرعت کلی)
1. روی تب **Performance** کلیک کنید
2. دکمه **Record** (دایره قرمز) را بزنید
3. صفحه را **Refresh** کنید (F5)
4. منتظر بمانید تا صفحه کامل لود شود
5. دکمه **Stop** (مربع) را بزنید
6. در پایین صفحه این اطلاعات را ببینید:
   - **Summary**: زمان کل
   - **Main**: زمان پردازش
7. این اعداد را یادداشت کنید 📝

### گام 5: تست Lighthouse (امتیاز کلی)
1. روی تب **Lighthouse** کلیک کنید
2. فقط **Performance** را انتخاب کنید
3. دکمه **Analyze page load** را بزنید
4. منتظر بمانید (30-60 ثانیه)
5. امتیاز Performance را ببینید (مثلاً 45/100)
6. این عدد را یادداشت کنید 📝

## مرحله 2: تست با Performance Monitor (کد ما)

### گام 1: اضافه کردن Performance Monitor به App
کد را اضافه می‌کنیم که خودکار اندازه‌گیری کند.

### گام 2: باز کردن Console
1. در DevTools، روی تب **Console** کلیک کنید
2. بعد از لود صفحه، این پیام را می‌بینید:
   ```
   📊 [Performance Monitor] Summary
   Page Load Time: 3200 ms
   Total API Calls: 15
   Average API Duration: 450 ms
   ```

## 🎯 اهداف بهبود

بعد از بهینه‌سازی باید این بهبودها را ببینید:

- ✅ تعداد Requests: کاهش 50-70%
- ✅ حجم کل: کاهش 30-50%
- ✅ زمان لود: کاهش 40-60%
- ✅ Performance Score: افزایش 20-40 امتیاز
- ✅ API Calls: کاهش 60-80%

---

## ⚠️ نکات مهم

1. **همیشه Cache را Clear کنید** قبل از تست
2. **از Network Throttling استفاده کنید** (در Network Tab → Throttling → Fast 3G) برای شبیه‌سازی اینترنت ایران
3. **چند بار تست کنید** و میانگین بگیرید
4. **در حالت Incognito تست کنید** برای اطمینان از عدم Cache

---

## 🔍 چطور بفهمیم کدام API کند است؟

1. در Network Tab، روی **Time** کلیک کنید (برای مرتب‌سازی)
2. API هایی که زمان زیادی دارند را پیدا کنید
3. روی هر API کلیک کنید و ببینید:
   - **Waiting (TTFB)**: زمان انتظار برای پاسخ سرور
   - **Content Download**: زمان دانلود داده

اگر **Waiting** زیاد است → مشکل از Backend
اگر **Content Download** زیاد است → مشکل از حجم داده
